## 概要

### 解くべき課題
ユーザー(いにわ)は楽天Koboで継続的にシリーズ物を購入している。新刊が出ても気づきにくく、発売から時間が経ってから買うことになる。そのため「購入済みシリーズの新刊を自動で教えてくれる」機能が欲しい。

### 解決策
- ユーザーが監視したい著者+タイトルキーワードを登録
- 1日1回、楽天ブックス書籍検索APIで新刊を探索
- 新刊発見時、楽天Kobo電子書籍検索APIでKobo版の存在を確認
- Discordで「新刊出たよ + Kobo版の有無 + 購入リンク」を通知

### スコープ(やらないこと)
- **購入履歴の自動取得**: 楽天Koboにユーザーライブラリ取得APIがないため、監視対象はユーザーが手動登録
- **発売前予告**: 楽天API経由の新刊情報は発売日ベース。予約開始時点の検知は将来拡張
- **複数書店の横断**: Amazon / honto / BookWalker は対象外
- **紙書籍の購入リンク**: 通知にリンクは含むが、紙購入誘導は副次的

### 非機能要件
- **可用性**: Pi(単一ノード)で動作、故障は受容
- **IPアドレス変動耐性**: 家庭回線の動的IP環境で運用するため、IP変動を検知して手動更新を促す
- **他ユニットへの影響**: 楽天API失敗で他機能が止まらないよう、サーキットブレーカーで隔離
- **通知スパム防止**: OBS配信中は通知を遅延、抑制ログを残す
- **登録直後の誤通知防止**: 登録時点で既刊を「既知」として backfill
